Binary Search on Answer
O(log(range) × isFeasible)
Why: The answer has a monotonic feasibility boundary — if X works, X+1 also works. Binary search finds the boundary in O(log(range)) calls to isFeasible.
Verify first: Write isFeasible(X). Confirm with two concrete examples: one where X is feasible and one where X is not. Verify the boundary is clean.

  if (isFeasible(mid)) hi = mid;

Option 2
1D Dynamic Programming
O(n) or O(n²)
Why: Subproblems overlap and depend on the sequence order. Each state dp[i] captures the optimal answer for a prefix ending at index i.
Verify first: Define dp[i] precisely. Ask: given only i and dp[0..i-1], can I determine dp[i]? If yes, state is complete.
Code shape ▼ 
vector<int> dp(n, INF);
dp[0] = base;
for (int i = 1; i < n; i++)
  for (int j = 0; j < i; j++)
    if (valid(j, i))
      dp[i] = min(dp[i], dp[j] + cost(j, i));
return dp[n-1];
⚠ Initialize dp with INF (minimize) or -INF (maximize) or 0 (count)
⚠ Define dp[i] as "ending AT i" vs "ending BY i" — they differ

These are the specific ways this approach fails — silent wrong answers and TLE conditions.
✗ isFeasible(X) is NOT monotone
Consequence: Binary search converges to wrong boundary — returns incorrect answer silently
Detect: Test: does isFeasible(X)=true guarantee isFeasible(X+1)=true? Verify with 3 examples.
Fix ▼
If not monotone — cannot use Binary Search on Answer. Reconsider approach.
✗ mid = (lo + hi) / 2 when lo + hi overflows
Consequence: Negative mid value → infinite loop
Detect: Check: lo and hi both close to INT_MAX?
Fix ▼
Always use: mid = lo + (hi - lo) / 2
